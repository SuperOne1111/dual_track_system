# 部署检查清单文档

## 部署前检查清单

### 系统环境检查
- [ ] 确认服务器操作系统版本兼容性
- [ ] 确认Python 3.10+环境已安装
- [ ] 确认所需依赖包已安装 (参考requirements.txt)
- [ ] 确认磁盘空间充足 (至少10GB可用空间)
- [ ] 确认内存容量满足要求 (推荐16GB+)
- [ ] 确认网络连接正常且防火墙配置正确

### 配置文件验证
- [ ] 检查`/workspace/config/production.yaml`文件存在且格式正确
- [ ] 验证数据库连接参数准确无误
- [ ] 确认API密钥和认证信息已正确配置
- [ ] 检查风险控制参数设置合理
- [ ] 验证数据访问日期边界设置正确 (防未来函数)
- [ ] 确认双轨策略参数独立配置 (趋势跟踪与均值回归)

### 数据库连接验证
- [ ] 测试数据库连接可达性
- [ ] 验证只读用户权限设置 (DB-02约束)
- [ ] 确认SELECT查询性能达标
- [ ] 验证查询超时设置合适
- [ ] 检查连接池配置参数

### 权限与安全检查
- [ ] 确认生产状态目录权限设置 (`/workspace/data/prod_states`)
- [ ] 验证日志目录写入权限
- [ ] 检查备份目录访问权限
- [ ] 确认敏感配置信息未硬编码
- [ ] 验证密码等敏感信息通过环境变量传递

### 合规性验证
- [ ] 验证无DDL操作 (仅SELECT语句) - DB-02
- [ ] 确认双轨隔离实现 - STRAT-01
- [ ] 验证防未来函数机制 - DB-03
- [ ] 确认脚本化运行模式 (无常驻进程) - RUN-01
- [ ] 验证原子写入机制 - RUN-02
- [ ] 检查逻辑一致性 (无模式分支) - CONS-02

## 部署执行步骤

### 1. 预部署验证
```bash
python scripts/deploy_prod.py
```

### 2. 环境准备
- [ ] 创建必要目录结构
- [ ] 设置正确的文件权限
- [ ] 初始化日志系统
- [ ] 验证原子写入功能

### 3. 配置部署
- [ ] 复制生产配置文件到指定位置
- [ ] 验证配置文件语法正确性
- [ ] 检查所有路径引用有效性
- [ ] 确认环境变量正确设置

### 4. 系统启动
- [ ] 执行系统启动命令
- [ ] 监控初始日志输出
- [ ] 验证各模块正常初始化
- [ ] 检查双轨策略正常运行

## 部署后验证

### 功能验证
- [ ] 确认趋势跟踪轨道正常运行
- [ ] 验证均值回归轨道正常运行
- [ ] 检查信号生成正常
- [ ] 验证风险管理功能
- [ ] 确认状态文件正确写入

### 性能验证
- [ ] 监控系统资源使用情况
- [ ] 验证查询响应时间符合预期
- [ ] 检查信号处理延迟
- [ ] 确认系统吞吐量达标

### 安全验证
- [ ] 验证无未授权访问
- [ ] 检查日志记录完整性
- [ ] 确认审计跟踪功能正常
- [ ] 验证数据保护措施有效

## 应急预案

### 回滚步骤
1. 停止当前生产实例
2. 恢复到上一个稳定版本
3. 重新配置并验证系统

### 监控告警
- [ ] 设置系统健康度监控
- [ ] 配置性能指标告警
- [ ] 确认异常通知机制有效
